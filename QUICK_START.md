# ForgeStudio M2 演示 - 快速开始

## 🚀 快速体验

### 方式一：导入预设 JSON（推荐）

1. 启动开发服务器（如果还没启动）：
   ```bash
   cd /Users/peak/work/pikecode/forgestudio
   cd apps/web && pnpm dev
   ```

2. 打开浏览器访问：http://localhost:5180/

3. 点击编辑器顶部的"导入 JSON"按钮

4. 选择项目根目录下的 `demo-product-detail.json` 文件

5. 页面将自动加载完整的商品详情页演示

### 方式二：手动构建

参考 `DEMO_PRODUCT_DETAIL.md` 文档，按步骤手动构建页面。

## 📱 演示功能

### 1. 商品信息展示
- ✅ 商品图片、名称、价格、描述
- ✅ 数据源绑定：`{{productData[0].name}}`
- ✅ 自动从 API 获取数据

### 2. 用户评论列表
- ✅ 循环渲染评论列表
- ✅ Switch 控制显示/隐藏
- ✅ 条件渲染：`showReviews`
- ✅ 列表项数据绑定：`{{$item.email}}`

### 3. 购买表单
- ✅ 姓名、电话输入框
- ✅ 数量加减按钮
- ✅ 表单提交到 API
- ✅ 成功/失败提示

### 4. 交互功能
- ✅ Switch onChange 事件
- ✅ Button onClick 事件
- ✅ Input onChange 事件
- ✅ Form onSubmit 事件

## 🎯 测试步骤

1. **查看商品信息**
   - 确认商品名称、价格、描述正确显示
   - 图片正常加载

2. **切换评论显示**
   - 点击评论区的 Switch 开关
   - 观察评论列表显示/隐藏

3. **调整购买数量**
   - 点击 "-" 按钮（最小值为 1）
   - 点击 "+" 按钮（无上限）
   - 观察数量实时更新

4. **提交订单**
   - 输入姓名和电话
   - 点击"提交订单"按钮
   - 查看成功提示（Toast）

5. **查看生成代码**
   - 点击右上角"代码预览"按钮
   - 查看生成的 Taro 代码
   - 验证包含：
     - useState 状态管理
     - useEffect 数据获取
     - 事件处理函数
     - 表单提交逻辑

6. **下载项目**
   - 点击"下载代码"按钮
   - 获取完整的 Taro 项目 ZIP 包
   - 解压后可直接运行

## 📊 技术指标

- **组件数量**: 24 个
- **数据源**: 2 个（商品 + 评论）
- **状态变量**: 4 个
- **事件处理器**: 5 个
- **循环渲染**: 1 个（评论列表）
- **条件渲染**: 1 个（评论区域）
- **表单字段**: 3 个
- **生成代码行数**: ~150 行

## 🎨 页面结构

```
ScrollView (滚动容器)
├── View (商品卡片)
│   ├── Image (商品图片)
│   ├── Text (商品名称)
│   ├── Text (商品价格)
│   └── Text (商品描述)
├── View (评论卡片)
│   ├── View (标题行)
│   │   ├── Text (标题)
│   │   └── Switch (显示开关)
│   └── List (评论列表) [循环] [条件]
│       └── Card (评论项)
│           ├── Text (评论者)
│           └── Text (评论内容)
└── Form (购买表单)
    ├── Text (表单标题)
    ├── Input (姓名)
    ├── Input (电话)
    ├── View (数量行)
    │   ├── Text (标签)
    │   ├── Button (减少)
    │   ├── Text (数量)
    │   └── Button (增加)
    └── Button (提交)
```

## 🔧 故障排除

### 问题：评论数据未显示
- 检查数据源 URL 是否正确
- 确认"自动获取"已勾选
- 查看浏览器控制台是否有错误

### 问题：表单提交失败
- 确认所有字段都已填写
- 检查网络连接
- 查看控制台错误信息

### 问题：代码生成不完整
- 确认所有组件都已正确配置
- 检查事件处理器是否正确绑定
- 重新构建项目
